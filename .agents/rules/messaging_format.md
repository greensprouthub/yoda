# Messaging Format Rule

## Rule
All messages sent to Baris — whether via WhatsApp broadcast or web chat — must use the SAME format so they look identical on both surfaces.

## Format Standard (WhatsApp-safe)

- **Bold:** Use `*text*` — renders as bold in WhatsApp AND looks clean in web chat
- **Sections:** Use emoji + ALL CAPS label instead of `##` headers (e.g. `📋 *WHAT GOT DONE*`)
- **Lists:** Use `•` bullet character or `–` dash, NOT markdown `-` or `*` for bullets
- **No markdown headers** (`##`, `###`) — ever in broadcast messages
- **No HTML** in messages
- **Line breaks:** Use blank lines between sections for breathing room
- **Max length:** Keep WhatsApp messages under ~300 words — split into follow-up if needed

## Template for Night/Session Wrap

```
🌙 *Yoda Wrap — [Date]*

✅ *DONE TODAY*
• [item 1]
• [item 2]

🔜 *NEXT UP*
• [item]

⚠️ *NEEDS YOUR INPUT*
• [item]

📊 *SYSTEM*
[brief stats line]
```

## Why
Baris works from both WhatsApp and the web. Discrepant formatting breaks continuity and causes confusion. One format, both surfaces, always.
